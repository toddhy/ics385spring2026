🗓️ Calendar Implementation Code Walkthrough

Honeymoon Booking System | React Frontend + Express Backend

📋 Overview

The calendar system handles guest bookings and admin date blocking for a honeymoon property. It features:

✓ Monthly calendar view with availability
✓ Guest booking & cancellation
✓ Admin-only date blocking
✓ Conflict detection
✓ Role-based access control
✓ Real-time booking updates

🔐 Security & Access Control

Authentication

  • All POST/DELETE routes require isAuthenticated middleware
  • GET /api/bookings is public (no auth required)

Authorization

  • blockDates: Admin-only (line 108)
  • Cancel booking: Owner or admin (line 160)
  • Book for visitor: Admin only (line 114)

Data Isolation

  • All queries filter by propertyKey
  • User snapshots immutable (prevent spoofing)

💡 Key Design Patterns

1. Normalized Date Keys
Uses "YYYY-MM-DD" strings for reliable date comparison across timezones.

2. User Snapshots
Stores user data at booking time (immutable record of who booked).

3. Soft Delete
Bookings marked 'cancelled' not removed (preserves history).

4. Conflict Detection
MongoDB $lte/$gte on start/end dates (overlapping range query).

5. Dual-Mode Calendar
Guest view (book own) vs. Admin view (book/block for visitors).
