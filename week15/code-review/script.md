5-Minute Code Review Script

0:00–1:00 — PRD Highlights
- Open PRD_v3.pdf. "Problem: [one-line]. Target: [one-line]."
- Show acceptance-criteria table and say how the project evolved (10–16 weeks).

1:00–2:00 — Live Demo
- "Now a quick demo." Show marketing page → dashboard (Chart.js + weather) → login → perform admin action → logout.
- Keep interactions minimal and predictable.

2:00–3:00 — Code Walk-through
- "I'll show `X` file." Explain intent, 2–3 key functions/lines, inputs/outputs, and error handling.

3:00–5:00 — Q&A
- Answer concisely. If unsure, say you would investigate and how.

Timing tips
- Practice with a stopwatch. Trim words to keep strictly to time.
- If a demo step fails, quickly explain and show the code instead.
