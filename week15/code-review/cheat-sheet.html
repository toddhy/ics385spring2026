  • Wk 5–9: Express routes, middleware, external APIs, error handling
  • Wk 12–13: React useEffect, Chart.js, data transforms
  • Wk 14–15: Passport local + Google, verify callback, session/JWT, route protection

Q&A Answers (Two lines each)

Why embedded reviews? Fast reads, simpler transactions. Use refs if reviews are large or shared.
Dashboard data fetch? useEffect calls /api/dashboard; server aggregates Mongo + OpenWeatherMap → JSON → Chart.js.
OAuth callback? Exchange code → read profile → find/create user in MongoDB → session/JWT → redirect.
Protected routes? Middleware checks isAuthenticated() or JWT → proceed to handler.
External API failures? Try/catch, timeouts, log, graceful fallback (cached data or UI message).

Contingency Phrases

  • "I'd investigate by checking logs, reproducing locally, and adding unit tests."
  • "I chose simpler schema for speed; at scale I'd migrate to referenced docs."
